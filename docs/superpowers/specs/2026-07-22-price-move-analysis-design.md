# 近期股价涨跌原因分析设计

## 目标

在每只配置股票的日报中增加独立的“近期股价涨跌原因”段落，并同步进入 Markdown、HTML 和飞书分段消息。内容必须明确区分已引用的事实、合理的市场联动推断和尚无法确认的原因，不能将大盘或行业表现表述为公司事实。

## 方案选择

1. 仅复用“前日表现与原因”的归因字段：改动小，但只能解释最近一个交易日，无法承载用户需要的近期走势分析，且与自动计算的日涨跌混在一起。
2. 新增结构化研究摘要字段：保留行情计算与研究归因的边界，能直接进入三个展示渠道，并让日常网页研究明确产出原因分析。采用此方案。
3. 自动从价格和新闻推导因果：可减少输入，但无法在缺少公司级证据时可靠区分相关性和因果关系，不采用。

## 数据契约

`StockResearchInput` 新增 `recent_price_move_summary`。为保证已保存的旧报告和旧输入仍可读取，该字段提供中文数据缺口默认值；新的日常研究提示要求每个标的显式填写，且内容应覆盖：

- 最近五个完整交易日或实际可验证窗口内的涨跌表现；
- 可引用的公司、行业、政策或消息驱动；
- 只属于合理联动的推断；
- 没有可验证原因时的明确说明和后续观察点。

该字段复用既有 `evidence` 来源清单，不新建未经验证的“因果事件”类型，也不影响推荐引擎的证据窗口与买卖建议。

## 展示与通知

报告将把该段落置于“前日表现与原因”之后、“基本面分析”之前。Markdown 渲染作为飞书的唯一内容来源，因此飞书的单公司分段也会自动包含该段落；HTML 模板以相同标题和内容呈现。研究输入缺失时显示中文缺口文本，绝不回退为英文模板文本。

## 验收标准

- 新字段可在新输入中验证，旧报告 JSON 仍可加载与重新渲染。
- Markdown、HTML 和飞书结构化分段均包含“近期股价涨跌原因”。
- 自动化提示要求证据边界和五日窗口，不允许将市场联动伪装成公司事实。
- 新增测试先失败后通过；全量测试、Ruff 检查、格式检查和 `git diff --check` 通过。
