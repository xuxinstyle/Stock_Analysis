# 股票研究与每日建议系统：设计规格

**日期：** 2026-07-21
**状态：** 待用户审核
**范围：** A 股、港股观察名单及可选持仓的每日研究系统；纳入美股与国际资讯作为外部影响因素。

## 1. 目标与边界

系统在中国时间每个交易日 09:00 生成开盘前研究报告，帮助用户对配置的 A 股和港股完成可追溯的基本面、行业、技术面、政策和消息面分析。每个标的同时给出短线、中线、长线的条件化研究建议。

报告是研究辅助材料，不保证收益、不构成个性化投资建议，也不执行或连接任何自动交易。所有建议必须给出依据、触发条件、失效/止损条件、风险等级和来源链接；无法验证的信息不得写成事实。

## 2. 用户场景

1. 用户在网页或配置文件中维护 A 股、港股的观察名单，并可选填持仓成本、数量、可用资金和风险偏好。
2. 每日 09:00，由 Codex 自动化任务读取配置、检索实时公开资料、分析前一交易日与隔夜事件，并生成研究报告。
3. 用户在网页仪表盘查看当日报告、单股详情和历史趋势，或以命令行手动立即执行同一流程。
4. 不填写持仓时，系统输出通用研究观点；填写持仓时，额外给出相对成本的盈亏、仓位与风险提示。

## 3. 支持的市场与研究时点

- **标的市场：** A 股与港股。
- **外部信息：** 美股、国际宏观、地缘政治、汇率、利率、大宗商品及与公司产品相关的价格变动。
- **运行时点：** 中国时间工作日 09:00。报告使用上一可得交易日的收盘行情、截至生成时的隔夜资讯和已发布公告。
- **节假日：** 根据市场交易状态分别标明 A 股、港股是否休市；无有效最新行情时使用最近交易日并显示日期，不伪造“昨日”数据。

## 4. 总体架构

采用 Python 单体应用，保证网页、命令行和自动化产物共用核心逻辑。

```text
Codex 定时自动化（09:00）
          │ 读取配置、网络检索与研究判断
          ▼
研究编排服务 ──► 行情/公告/资讯适配器 ──► 证据归档
          │               │
          │               └── 技术指标、异动、去重与可信度计算
          ▼
建议规则与报告模型
          ├── JSON 报告与运行记录（SQLite/文件）
          ├── Markdown 审阅版
          └── HTML 打印版
                    │
     FastAPI 网页仪表盘 / Typer 命令行
```

### 4.1 技术选型

- **后端与网页：** Python + FastAPI，服务仪表盘、配置页面、报告和历史查询接口。
- **命令行：** Typer，支持初始化、校验配置、立即生成报告、查看最近报告和启动网页。
- **持久化：** SQLite 存储配置元数据、运行状态、报告索引和结构化分析结果；完整 Markdown/HTML/JSON 以文件归档。
- **配置：** YAML 作为可版本控制的初始配置来源；网页表单修改后写入受校验的持久化层。
- **指标与渲染：** 独立的 Python 领域模块完成价格计算、指标、建议规则和多格式报告渲染。

## 5. Codex 自动化与人工研究

每日运行使用 Codex 应用的本地项目自动化，直接由 Codex 执行网络检索、资料比对、事件归因与报告文字撰写，不要求用户另行提供模型 API 密钥。

项目中的确定性代码不假定能够自行调用 Codex 或网络搜索工具。它负责输入校验、数据结构、技术指标、建议边界、报告渲染、历史管理与异常处理。自动化任务把已检索的证据和结构化结论写入这些稳定接口；这使网页与 CLI 即使没有运行自动化，也能可靠地浏览和复现已有报告。

若日后需要在没有 Codex App 的服务器上部署，可增加模型与搜索提供商适配器，但不属于首版范围。

## 6. 数据与证据策略

### 6.1 采集类别

每只标的至少尝试收集：

1. 前一交易日及近期的价格、成交量、涨跌幅、波动和可用历史 K 线；
2. 公司公告、财报、业绩预告、投资者关系材料和交易所披露；
3. 行业供需、景气度、竞争格局、上下游变化，以及主营相关产品的价格快速上涨或下跌；
4. 国内政策、监管、税费、产业支持与限制；
5. 公司新闻、市场传闻的可验证部分、分析师观点和重大事件；
6. 美股同行、海外需求、国际宏观、汇率、利率、地缘事件及大宗商品变化。

### 6.2 证据记录与冲突处理

每条证据记录保存标题、URL、来源名称、发布时间、抓取时间、涉及标的/行业、方向、可信度与摘要。优先级为交易所/公司/政府/监管等一手来源，其次为信誉媒体与行业机构；无法确认来源、未提供发布时间或相互矛盾的内容降低可信度。

新闻结论必须能回链至少一条来源。重大单一消息不能直接触发强烈建议，除非它来自一手来源或存在独立确认。资料不足时报告应写明“数据不足/待验证”，而非推断为利多或利空。

## 7. 分析框架

### 7.1 前日表现与技术面

对前一交易日涨跌、成交量、波动、相对行业表现及可获取的资金/北向等公开指标作归因。技术层至少计算并解释均线趋势、RSI、MACD、布林带、成交量异常、支撑/阻力和近期波动；任何指标都以历史数据完整性为前提。

### 7.2 基本面与行业

基本面覆盖业绩增长、盈利能力、现金流、估值、负债与已披露经营变化。行业层覆盖需求、供给、竞争、上下游、产品价格和行业政策。对产品价格或行业指标的快速变动，说明变化幅度、时间、可能传导路径和待验证环节。

### 7.3 政策、消息与外部传导

政策层说明发布主体、适用范围、实施时间、直接影响和不确定性。消息层分别归类公司、行业、国内市场、美股与国际事件；美股和国际内容需说明它为何影响该标的，而不是只罗列新闻。

### 7.4 突发异动

将以下情形标为突发事件：显著价格/成交量异动、重大公告、监管行动、财报超预期或不及预期、关键产品价格快速变化、重要海外传导事件。每项说明发生时间、证据、潜在影响、可信度和下一步待确认信息。

## 8. 建议引擎

每只股票按短线（数日到数周）、中线（数周到数月）和长线（半年以上）分别输出：

- 研究动作：观望、关注、分批买入、持有、减仓或回避；
- 主要依据：清晰区分事实、推断和观点；
- 生效触发：价格区间、技术确认、公告/行业数据验证或事件条件；
- 目标/观察区间与时间框架；
- 止损或失效条件；
- 仓位上限与风险等级。

持仓信息存在时，建议还会引用持仓成本、浮盈亏和用户风险偏好；不存在时不臆测用户资金状况。建议由确定性风险约束限制，例如资料不足、来源冲突或高波动时不能输出高确信度的积极操作结论。

## 9. 每日报告结构

报告开头列出生成时间、覆盖市场、数据截止时间、休市状态、全局风险摘要和本次失败/缺失来源。随后按配置顺序列出每只股票：

1. 标的身份与前一交易日表现及归因；
2. 基本面分析；
3. 行业分析与产品价格；
4. 技术面分析；
5. 政策分析；
6. 消息面分析（公司、国内、美股与国际传导）；
7. 突发事件和待验证事项；
8. 短/中/长线条件化建议；
9. 持仓影响（仅在已配置时）；
10. 来源链接、可信度和数据缺口。

同一份内容导出为 JSON（机器读取）、Markdown（审阅）、HTML（网页和打印）。网页提供当日概览、单股详情、配置维护、运行状态和历史报告入口。

## 10. 命令行与网页能力

命令行首版提供：初始化样例配置、校验配置、立即运行日常流程、列出报告、查看指定报告，以及启动网页服务。网页首版提供股票配置增删改、可选持仓输入、最近报告概览、报告详情与历史浏览。网页不提供券商连接或自动下单。

## 11. 失败降级与运行记录

- 行情或单个来源不可用时，继续使用其余来源并在报告中注明范围和时间戳；
- 配置无效、数据为空或解析失败时，记录阶段、原因和可复现的输入，不产生虚假结论；
- 市场休市、历史数据不足或资讯未验证时，明确显示限制；
- 自动化每次运行写入开始/结束时间、步骤状态、错误、输出路径和报告版本；
- 同一资料只保留一次证据记录，重复报道不放大信号；
- 任何未处理异常都使运行记录标记失败，并保留已有、明确标为不完整的产物。

## 12. 验收与测试

### 单元测试

- YAML/网页配置、股票代码与可选持仓字段验证；
- 均线、RSI、MACD、布林带、成交量和价格异动计算；
- 建议的触发、风险降级和持仓边界规则；
- 证据去重、可信度评级和来源格式化；
- JSON、Markdown、HTML 报告字段完整性与一致性。

### 集成测试

使用固定的行情、公告和新闻样本运行完整流水线，验证每只股票生成全部章节、来源可追踪、网页与 CLI 读取同一份报告。针对休市、无历史数据、来源冲突、仅部分抓取成功和突发事件分别建立样本。

### 人工验收

通过 CLI 与网页各执行一次样例报告；检查每日自动化能够在 09:00 生成产物并留下运行记录；人工核对至少一个标的的新闻链接、前日行情和建议触发条件。

## 13. 非目标

- 不执行交易、不连接券商账户；
- 不承诺收益、不输出无依据的确定性预测；
- 首版不要求用户配置或付费购买模型 API；
- 首版不部署为无 Codex App 依赖的云端服务；
- 首版不承诺覆盖未在配置中的全部股票。

## 14. 实施顺序

1. 建立 Python 项目、配置模型、存储结构、领域类型和测试骨架；
2. 实现行情/证据接口、指标、建议约束和多格式报告；
3. 实现 CLI、网页仪表盘及固定样本的端到端流程；
4. 创建 Codex 每日 09:00 自动化，并用样例配置进行一次人工验证；
5. 补充运行日志、异常展示和历史浏览。
